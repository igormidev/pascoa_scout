# Pascoa Scout

Serverpod workspace for Pascoa Scout.

This repository now has a standard Serverpod layout with three projects:

- `pascoa_scout_server`
  The Serverpod backend.
- `pascoa_scout_client`
  The generated Dart client for talking to the backend.
- `pascoa_scout_flutter`
  The existing Flutter app from this repository, moved into the Serverpod workspace instead of reusing the Flutter app generated by `serverpod create`.

## Why The Flutter App Was Moved

The original repository already had the macOS Flutter setup you wanted to keep. The default Serverpod scaffold generates a full Flutter app with Android, iOS, web, Linux, Windows, and macOS targets. To preserve the existing macOS-only app setup, the generated Flutter folder was deleted completely and the current app was moved into `pascoa_scout_flutter`.

## Current Flutter App Notes

- The Flutter app keeps the existing Apify / Upwork scraping implementation.
- Repository methods use `result_dart` with `PascoaException` on the failure side.
- The Flutter app now lives inside the Serverpod workspace and can start integrating `pascoa_scout_client` and `serverpod_flutter`.

## Workspace Notes

- Root `pubspec.yaml` is the Serverpod workspace manifest.
- Run dependency resolution from the workspace root.
- Keep generated files in sync when changing Freezed or Serverpod protocol/model definitions.
